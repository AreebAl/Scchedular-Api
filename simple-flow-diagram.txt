API ENDPOINTS:
==============
1. AMSP API: GET /amsp/api/masterdata/v1/sites
   Response: Array of Site objects (id, name, city, clusterName, etc.)

2. Provisioning API: GET /sps/v1/site?SiteName={SiteName}
   Response: ProvisioningSite object (Site, CM, Ranges, Extensions)

COMPONENTS:
===========
- AMSP Client: HTTP client for AMSP API
- Site Service: Business logic for site operations
- Site Controller: REST endpoints
- Provisioning Service: Business logic for provisioning
- Provisioning Client: HTTP client for Provisioning API
- Error Handler: Centralized error management
- Admin Portal: User interface
